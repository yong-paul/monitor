
# 本地监控连接不上云端服务 - 故障排除指南

## 问题分析

根据代码分析，当前配置如下：

**本地监控 (monitor_service.py):**
- 云端 Redis 主机: `8.153.77.252`
- 云端 Redis 端口: `6378`
- 云端 Redis 密码: `7712`

**云端服务 (app.py):**
- Redis 主机: `172.18.0.4` (这是 Docker 容器内部 IP)
- Redis 端口: `6378`
- Redis 密码: `7712`

## 可能的原因及解决方案

### 原因 1: 云服务器防火墙未开放 6378 端口

**解决方法:**

#### 如果使用阿里云/腾讯云等云服务商：
1. 登录云服务器控制台
2. 找到「安全组」或「防火墙」设置
3. 添加入站规则：
   - 协议端口: `TCP:6378`
   - 授权对象: `0.0.0.0/0` (或你的本地 IP，更安全)
   - 策略: 允许

#### 如果使用 iptables (Linux):
```bash
# 开放 6378 端口
sudo iptables -I INPUT -p tcp --dport 6378 -j ACCEPT
# 保存规则
sudo service iptables save
```

#### 如果使用 firewalld (CentOS/RHEL):
```bash
sudo firewall-cmd --permanent --add-port=6378/tcp
sudo firewall-cmd --reload
```

#### 如果使用 ufw (Ubuntu/Debian):
```bash
sudo ufw allow 6378/tcp
sudo ufw reload
```

---

### 原因 2: Redis 仅监听本地地址 (127.0.0.1)

**检查方法:**
在云服务器上运行：
```bash
netstat -tlnp | grep 6378
```

如果看到 `127.0.0.1:6378` 或 `::1:6378`，说明 Redis 只监听本地。

**解决方法:**

1. 编辑 Redis 配置文件 (通常是 `/etc/redis/redis.conf` 或 `/etc/redis.conf`)
2. 找到 `bind` 配置项，修改为：
   ```
   bind 0.0.0.0
   ```
   或者注释掉这一行
3. 找到 `protected-mode` 配置项，修改为：
   ```
   protected-mode no
   ```
4. 重启 Redis 服务：
   ```bash
   sudo systemctl restart redis
   # 或者
   sudo service redis restart
   ```

---

### 原因 3: Redis 服务未启动

**检查方法:**
在云服务器上运行：
```bash
ps aux | grep redis
# 或者
systemctl status redis
```

**解决方法:**
```bash
# 启动 Redis
sudo systemctl start redis
# 或者
sudo service redis start
# 设置开机自启
sudo systemctl enable redis
```

---

### 原因 4: 使用 Docker 部署 Redis 但端口映射不正确

如果你的 Redis 是通过 Docker 部署的，需要确保端口映射正确。

**检查方法:**
```bash
docker ps
```

**解决方法:**
确保运行 Redis 容器时正确映射了端口：
```bash
docker run -d -p 6378:6379 --name redis redis:latest
# 注意: 容器内部是 6379，映射到宿主机的 6378
```

或者如果使用 docker-compose：
```yaml
services:
  redis:
    image: redis:latest
    ports:
      - "6378:6379"
    command: redis-server --requirepass 7712
```

---

### 原因 5: 密码验证失败

**检查方法:**
在云服务器上本地测试 Redis 连接：
```bash
redis-cli -h 127.0.0.1 -p 6378 -a 7712 ping
```

如果返回 `PONG`，说明密码正确。

**解决方法:**
确保 `monitor_service.py` 和 `app.py` 中的密码配置一致，都是 `7712`。

## 常见问题

### Q: 为什么 app.py 中使用的是 172.18.0.4 而不是 8.153.77.252？

A: `172.18.0.4` 很可能是 Docker 内部网络 IP。因为 `app.py` 和 Redis 都在同一台云服务器上，它们可以通过内网 IP 通信，这样更安全且速度更快。

### Q: 如何确认 monitor:status 键是否被写入？

A: 在云服务器上执行：
```bash
redis-cli -h 127.0.0.1 -p 6378 -a 7712 GET monitor:status
```

如果返回 JSON 数据，说明本地监控服务成功上报了数据。

### Q: 可以用其他端口吗？

A: 可以，但需要确保：
1. 云服务器防火墙开放该端口
2. Redis 监听该端口
3. `monitor_service.py` 和 `app.py` 中的配置同步修改
